EverEdit支持完备的查找和替换功能，您可以使用EverEdit的查找替换完成各种文本整理、整合、过滤等功能。

#1. 查找

##增量查找
当选择增量查找时，随着您在输入框中的输入，会以光标位置为起始点逐一向下进行匹配。

##扩展(\n,\r,\t)
主要用于普通查找替换时对于常见的回车、换行、制表符的简化处理。当勾选这个时，\n将会被认为是换行符，同理\t会被认为是制表符。如果需要查找这些字符串本身，应该使用\进行转义。
比如查找\n字符串本身的话，应该使用\\\n。注意：此复选框和正则表达式复选框是互斥的。

##高亮匹配
勾选此复选框时，进行查找的时候，会在可视区域用显著地背景色把所有的匹配标记出来。

##继续到下一个文件
当您希望在所有打开的文件中查找但不希望主动点击tab进行切换时，可以勾选此复选框。


##查找全部
查找全部会把所有的匹配按照行的形式都输出到输出窗口中，在输出窗口中可以双击进行跳转。


##高级操作
###过滤匹配
过滤匹配仅在勾选正则时有意义。比如我们需要列出文档中包含的所有数字，那么可以输入\d+，然后查找全部即可。

###筛选行
当该行有字符串匹配时，那么把该行字符串输出到输出窗口中。

###统计匹配数
计算共有多少个字符串符合匹配。

###设置书签
为每一个匹配的行设置上书签。

###在所有打开的文件中查找全部
这个是查找全部的增强版，顾名思义，就是在所有打开的窗口中，搜寻匹配，并把所有的匹配显示到输出窗口中。

#2. 替换

##以高速模式进行替换
当前打开的文档有着复杂的数据结构，在进行大批量替换时，为了维护这些数据结构的关系，浪费了大量的CPU时间。高速替换就是为了解决这个问题而来。

高速替换相当与把当前buffer看成一个整体，一个连续的内存空间；也就是说您可以在这里使用任意的正则表达式，包括各种跨行的正则。EverEdit均可在常量时间内完成替换操作。
如果您需要对一个较大的文档进行数百万次替换的话，建议使用该功能。

##范围
替换操作执行的范围。如勾选选区的话，那么替换仅在选区内进行。

#3. 关于多行查找的问题
EverEdit在正则模式下，因性能问题对多行支持不完美，不过仍然有能力支持指定个数的简单的多行查找。比如我们如果要查询连续2个换行符的话，应该使用\n\n，而不应该使用\n{2}这种不定个数的查找。
在普通模式下，因\n始终是明确指定的，所以没有问题。

在用正则进行替换的时候，EverEdit也有类似的问题，需要明确指定\n；不过您可以使用高速替换，在高速替换下，替换所使用的正则不受限制。

#4. 关于环视(正向、反向预搜索)
因正则引擎限制，在进行替换时，EverEdit不支持使用环视正则表达式。不过这个可以用分组来解决，比如我们替换(?<=ever)edit这个时，可以使用(ever)edit来解决。

#5. 批量替换
所谓批量替换就是在同一个文档中反复的进行多次替换，EverEdit提供了GUI工具用于完成这种复杂的工作。主菜单→查找→批量替换。

##新建替换项
点击下方的加号会新建一个替换，并置于选中状态。在上方的列表区域双击不同的部分，将会弹出输入框，输入各种内容。输入完毕之后，务必按下回车予以确认，否则输入的内容将会被抛弃。

如需修改响应的替换项的内容，双击在弹出的输入框中修改，然后回车确定即可。

**技巧:**快捷键Ctrl+1/Ctrl+2/Ctrl+3分别对应着列表的第一~三个内容。

##删除替换项
如需删除替换项目,选中该替换项目，然后按下Delete键或者点击下方的X按钮。**注意:**所有处于选中状态的文本都会被删除。

如果仅仅希望该替换项目在这次操作的时候不起作用，但不希望它被删除，因为可能在下次替换中会使用。可以到该替换项中，把前面的勾去掉即可。

##更改替换的项的替换设置
每一个替换项都有对应的选项，有的是正则替换，有的需要区分大小写。在对应的替换项中，右键单击，会弹出相应的菜单，您可以在菜单中勾选，然后选项中会出现用字母表示的设置。

字母表示的意义如下:

```
C:区分大小写
R:正则替换
W:全字匹配
X:扩展\n\t\r为对应的回车换行制表符
```

##进行部分替换
在当前的列表中，有时候我们仅仅使用其中的一部分替换项，这个时候可以选中需要的替换项，然后点击替换所选按钮，完成部分替换的目的。

##调整替换顺序
在很多替换中，替换项的前后顺序非常重要。用户可以使用向上、向下进行简单的调整。

##导出替换表
批量替换的所有项目可以被导出为非常简单的tsv文件。导出的文件中，每一行代表一个替换项。行的格式如下:

```
on	text	find	replace	option
```

每一列的意义如下:

```
on:表示该项是否被勾选，如未被勾选则为off
text:表示这是一个文本替换，目前请始终设置为text；将来可以通过插件扩充更多的模式，比如繁简转换等
find:查找的文本
replace:替换为的文本
option:选项
```

##导入替换表
EverEdit同样支持导入tsv格式的替换表。用户可以在EverEdit中用文本编辑模式整理出大量的替换表然后保存为tsv，在批量替换中导入即可。所有被导入过的文件都被记录下来，可以单击批量替换界面中的星号*按钮，在弹出菜单中选择历史文件。

#6. 在文件中查找和替换
很多时候我们需要查找或替换某个特定文件夹下指定后缀的文件，EverEdit对此提供了完美的支持。按下Ctrl+Shift+F即可调出对话框。

###文件夹的路径
默认的情况下，对话框中的文件夹路径始终定位到当前文件所在的目录，如果不希望这个功能的话，可以到常规选项→[在文件中查找时使用当前文件路径]中设置。

###仅显示文件路径
非常方便的只显示出包含特定字符串的文件的全路径。

###使用输出窗口
默认的情况下，查找结果会被重定向到一个文件，在查找结束时显示该路径。[使用输出窗口]会把所有的查找结果输出到输出窗口中。

###锁定主窗口
推荐使用该设置，以避免潜在的多线程问题带来的崩溃。除此之外，锁定主窗口会使查找的效率更高。

###备份被替换的文件
在文件夹中进行替换是个较为危险的操作，因为这儿操作是不可逆的，做好适当的备份是个聪明的选择。勾选此选项的话，EverEdit在替换时会在对应的文件路径下生成同名但后缀为.bak的备份文件，以防不测。

###编码
默认的情况下，EverEdit将会自动探测文件的编码。但自动探测难免发生失败，在失败的情况下，您可以强制选择编码。

#7. 万能查找
EverEdit内置的万能查找，同样对当前文件的文本查找和替换提供了完美的支持。具体请参考**万能查找**的设置。
